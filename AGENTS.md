# 项目级编码约束

## 1. 项目目标与当前范围

本项目为“办公室资产租借管理与可视化运营系统设计”。

当前 V1 正式实现范围仅包括以下核心链路：

- Borrow（借出确认）
- Return（归还确认）

以下内容在当前阶段不属于正式落地范围，禁止擅自扩展为主链路实现：

- Inbound（入库正式流程）
- 预约
- 续借
- 验收
- 盘点
- 多角色复杂权限体系
- 完整看板统计实现
- 完整 Web API 层
- WebSocket 完整推送体系

可以为后续扩展预留接口，但不得把上述内容硬塞进当前 V1 核心闭环。

---

## 2. 当前仓库中的模块分层

### 串口通信层
目录：
- `serial_comm/`

该层负责：
- 协议帧结构
- 编解码
- CRC 校验
- ACK 匹配
- ACK 重传
- 心跳检测
- 去重
- 串口收发
- Mock MCU 联调支持

当前串口通信层已具备 V1 可运行实现，默认视为稳定模块。

### 后端服务层
目录：
- `后端服务/`

该层当前处于 V1 雏形阶段，后续编码工作主要发生在这一层。

## 3. 串口通信层默认禁止改动

未经明确批准，不要修改以下文件的核心行为：

- `serial_comm/protocol.py`
- `serial_comm/serial_manager.py`
- `serial_comm/mock_mcu.py`

如果确需改动，只允许最小必要修改，不得破坏以下既有语义：

- 帧结构
- SeqID 机制
- Big-Endian 约定
- CRC16-CCITT 校验
- ACK 匹配
- ACK 重传
- 心跳检测
- 最近 5 个 Seq 去重
- request_seq 与发送 seq_id 对齐语义

不得重写或推翻现有串口通信层实现。

## 4. 服务层职责边界

### 服务层负责
- Borrow / Return 流程编排
- 调用规则层
- 创建待确认事务
- 发起硬件确认请求
- 等待事务推进结果
- 触发数据库原子提交
- 构建统一 BusinessResult

### 服务层不负责
- 串口协议封包/拆包
- CRC 校验
- ACK 重传
- 心跳检测
- 底层串口连接管理
- SQL 细节
- 前端展示逻辑
- 视觉识别算法

---

## 5. 当前必须遵守的事务状态机

正式状态链路固定为：

- WAIT_ACK
- WAIT_HW
- UPDATING
- COMPLETED / FAILED

必须满足：

- 只有 ACK_OK 后才能进入 WAIT_HW
- 只有收到合法匹配且 confirm_result == CONFIRMED 的 EVT_USER_ACTION 后才能进入 UPDATING
- 只有原子提交成功后才能进入 COMPLETED
- 任一失败路径进入 FAILED
- 非法事件、重复事件、孤儿事件不得推进状态

---

## 6. 硬件事件匹配规则

不得仅凭 `asset_id` 匹配硬件事件。

必须保持以下联合匹配规则：

- `asset_id`
- `request_seq`
- `action_type`
- `request_id`

如果事务带有 `request_id`，则硬件回调必须带回该值，且完全一致。

以下事件必须忽略，不得推进事务：

- 缺字段事件
- request_seq 不匹配事件
- action_type 不匹配事件
- request_id 不匹配事件
- 重复事件
- 孤儿事件
- 迟到事件

---

## 7. 数据库与原子提交约束

关键业务流水表必须保留硬件追溯字段：

- `hw_seq`
- `hw_result`

必须保证：
- 资产状态更新
- operation_records 写入

处于同一原子边界。

不得出现以下半成功状态：

- 资产状态已更新，但流水未写入
- 流水已写入，但资产状态未更新

---

## 8. 当前正式资产状态

当前正式资产状态仅为：

- IN_STOCK
- BORROWED
- MAINTENANCE
- SCRAPPED

不得擅自新增新的正式资产状态并写入主链路。

## 9. 当前编码任务

当前阶段仅允许进行以下工作：

- 抽离 `RuleService`
- 抽离 `TransactionManager`
- 收敛 `AssetConfirmService` 为编排器
- 补充测试
- 在不破坏串口通信层的前提下完善 Borrow / Return 的 V1 后端核心闭环

不得在本轮任务中擅自扩展为：
- 完整 Web 后端
- 完整权限系统
- 新的硬件协议
- 入库正式闭环
- 预约/续借等附加业务
